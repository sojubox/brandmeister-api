RESTful API server endpoint documentation ("the API docs")

This API is RESTful. That means this API follows six basic design patterns or rules:

  1. Client-Server:
    They are really separated layers. Client's behavior is only to consume resources from this Server.

  2. Stateless:
    Each Client's request must contain all information required by the Server to carry out a proper response. In other words: the Server won't store Client's information from one request to the next.

  3. Cacheable:
    Server will indicate the Client if something is cacheable or not.

  4. Layered System:
    RESTful API design allows intermediaries to operate among Clients and Server.

  5. Uniform interface:
    This RESTful API uses a standard and uniform method for Client-to-Server communication.

  6. Code-on-Demand:
    Server might provide executable code or scripts for Clients to execute in their context.
